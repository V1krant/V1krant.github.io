Control flow and Loops

A quote from MDN Web Docs I think is great describing the control flow which is how your computer reads the code you've given it which is top to bottom and anything in between that can change the flow a bit eg. going from bottom back to top again, but will always go from top to bottom. "Code is run in order from the first line in the file to the last line, unless the computer runs across the (extremely frequent) structures that change the control flow, such as conditionals and loops."
