Javascript and the DOM

The Javascript and the DOM is got to be for me at least the most fascinating things I've learned about so far. The DOM is the file structure of a website and with this, we can "look over" the site and get a sort of bird's eye view of what's going on and what the contents are of the website. This is awesome when we couple this with Javascript now we can start to inject things, toggle things in and out, delete things. We have do have full control over the DOM and all its contents with Javascript. Imagine being able to delete a blog post underneath the viewed post you're looking at and place posts, ads, information underneath…. and it's not far of a stretch to think how some tech giants implement algorithms that pull and place things into your feed they think you like.

Arrays and Objects

This brings me to how do we store that information and pull it out when we need it. One way is to use an array which you can use a word to store lots of information, you can imagine this by thinking of it as a sort of a shopping list inside of brackets and as you go down the list you count down from 0 being the first and the last number being whatever is last on your shopping list which is great because we can store and pull so many items in it! But in arrays we store information that is the “same of its kind”, like food we have here! Now if we wanted the 10th item on the list all we’d have to do is put in our grocery list name and the number 9. Sadly in programming, we count from 0 haha. Now there is another way we can store these and it is with objects! They are similar, but instead of pulling information out with the word or value then a number, you have to state the word and then a keyword you have given to your “value” or information.
